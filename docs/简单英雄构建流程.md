# 简单英雄构建流程

## 逻辑概述

AllPlayers类下有

> public List<Hero> heroes;

具体内容请自行查阅，里面存储的核心内容即hero.heroname

后在BeforeAttack和AfterAttack中添加对应内容

(注：instance为玩家动作选择后，描述实际造成效果的类，对其的修改就是对最终效果的修改)
